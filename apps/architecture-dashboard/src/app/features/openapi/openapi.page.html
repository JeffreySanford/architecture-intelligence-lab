Generated models vs local app shape
How generated DTOs are consumed by Angular state

Generated clients expose typed models derived from Spring and Nest OpenAPI contracts. Angular facades map those generated DTOs into local application ViewModel shapes, so the UI layer remains decoupled from raw contract shape changes.

This boundary protects the app from contract drift by keeping generated model changes localized to the facade and store layers before they reach the page views.

Performance guidance
Filter and render latency for large contract lists

The OpenAPI contract dashboard is designed to use pagination and client-side filtering. Large generated client or endpoint lists should avoid rendering the full contract set at once.

For very large contract inventories, consider virtual scrolling or server-side filtering to keep route transition and table render latency low.
